# 在庫入出庫管理システム

社内の在庫入出庫を掲示板形式で管理するWebアプリケーションです。依頼者が商品の出庫依頼・入庫要求を掲示し、発注者がそれに対応して発注を行います。

## 主な機能

- **依頼掲示板**: 出庫依頼・入庫要求を一覧表示。商品名・拠点であいまい検索可能
- **複数商品登録**: 1依頼につき最大10商品。商品コード、商品名、数量、単位（ケース/本）、賞味期限を記録
- **発注機能**: 発注者名・内線番号・発注数量・備考を入力して発注。依頼数を超える発注は不可
- **自動ステータス管理**: 全商品の発注数量が満たされると自動で「完了」に変更
- **メール通知**: 完了時に依頼者のメールアドレスへ自動通知（SMTP設定が必要）
- **PIN認証**: 依頼追加時に設定した4〜6桁のPINで編集・削除を制御
- **ダークモード**: ライト/ダーク切り替え対応

## 技術スタック

- **フロントエンド**: React + Vite + Tailwind CSS + shadcn/ui
- **バックエンド**: Express.js + better-sqlite3 + Drizzle ORM
- **データベース**: SQLite（ファイルベース、設定でパス変更可）
- **メール送信**: nodemailer（SMTP設定で有効化）

## セットアップ

### 1. 依存パッケージのインストール

```bash
npm install
```

### 2. 環境変数の設定

`.env.example` を `.env` にコピーして編集します。

```bash
cp .env.example .env
```

#### 必須設定なし（デフォルトで動作）

| 変数 | 説明 | デフォルト |
|------|------|----------|
| `DB_PATH` | SQLiteファイルのパス | `./zaiko.db` |
| `PORT` | サーバーポート | `5000` |

#### メール送信（任意）

SMTP設定を行うと、依頼完了時にメール通知が送信されます。

| 変数 | 説明 |
|------|------|
| `SMTP_HOST` | SMTPサーバーのホスト名 |
| `SMTP_PORT` | SMTPポート（587 or 465） |
| `SMTP_USER` | SMTP認証ユーザー |
| `SMTP_PASS` | SMTP認証パスワード |
| `SMTP_FROM` | 送信元メールアドレス |

### 3. 開発サーバーの起動

```bash
npm run dev
```

ブラウザで `http://localhost:5000` にアクセスしてください。

### 4. プロダクションビルド

```bash
npm run build
npm start
```

## 使い方

### 依頼者（商品を掲示する人）

1. 「新規依頼」から出庫依頼または入庫要求を作成
2. 商品情報（コード、名前、数量、単位、賞味期限）を最大10件入力
3. 依頼拠点、依頼者名、内線番号、メールアドレス、PINを設定
4. 掲示期限を設定して投稿

### 発注者（発注する人）

1. 掲示板から対応したい依頼をクリック
2. 詳細画面で「発注する」ボタンを押下
3. 発注数量、発注者名、内線番号、備考を入力して確定

### ステータスの流れ

- **受付中** → 発注待ち
- **対応中** → 一部発注あり
- **完了** → 全数量の発注完了（自動切り替え + メール通知）

## Render（無料）へのデプロイ手順

[Render](https://render.com) は無料でNode.jsアプリをホスティングできるサービスです。

### 手順

1. [render.com](https://render.com) でアカウントを作成（無料）
2. ダッシュボードの **「New +」→「Web Service」** をクリック
3. **「Connect a repository」** でGitHubアカウントを連携し、`zaiko-kanri` リポジトリを選択
4. 以下の設定を確認（`render.yaml` が自動で読み込まれるので基本そのままでOK）：

   | 項目 | 値 |
   |------|----|
   | Runtime | Node |
   | Build Command | `npm install && npm run build` |
   | Start Command | `node dist/index.cjs` |
   | Plan | **Free** |

5. **「Create Web Service」** をクリック → 自動でビルド・デプロイが始まります
6. 完了すると `https://zaiko-kanri-xxxx.onrender.com` のようなURLが発行されます

### データの保存について

`render.yaml` に **Disk（1GB）** の設定が含まれています。これによりSQLiteのデータがサーバー再起動後も消えずに保存されます。

> ⚠️ Renderの無料プランはアクセスがない時間帯にサーバーがスリープします。初回アクセス時に起動に数秒かかる場合があります。

### メール通知を使いたい場合

Renderのダッシュボード → サービス → **「Environment」** タブで以下を追加：

| 変数名 | 内容 |
|--------|------|
| `SMTP_HOST` | SMTPサーバーホスト名 |
| `SMTP_PORT` | `587` など |
| `SMTP_USER` | メールアドレス |
| `SMTP_PASS` | パスワード |
| `SMTP_FROM` | 送信元アドレス |
